# AI 社群助教系统 · 技术实现详解

> 子默出品 · 2026-03-30
> 目标：给初序微信社群部署 AI 健康顾问，具备永久记忆 + 体质追踪

---

## 系统是什么？

**一个微信群里的 AI 助教**，24小时在线，能做到：

| 功能 | 说明 |
|------|------|
| 💬 智能问答 | 用户在群里 @AI 提问，AI 基于健康知识库回答 |
| 🧠 永久记忆 | 记住每个用户的所有对话，知道张三是"脾虚体质" |
| 📊 体质追踪 | 根据用户过往对话，追踪健康变化 |
| 📢 主动推送 | 每天定时推送个性化养生建议 |
| 🚨 专家转介 | 复杂问题自动标记，通知管理员/专家介入 |

## 系统架构（一张图看懂）

```
用户在微信群 @AI 提问
        ↓
┌──── 消息接入层 ────┐
│  企业微信 API       │  ← 唯一合规渠道（2026年）
│  接收消息+发送回复  │
└────────┬───────────┘
         ↓
┌──── 大脑层（核心）──┐
│                      │
│  ① 用户识别         │  ← 这是谁？查用户库
│       ↓              │
│  ② 记忆调取         │  ← 他之前说过什么？查记忆库
│       ↓              │
│  ③ 知识检索（RAG）  │  ← 从健康知识库找相关内容
│       ↓              │
│  ④ AI 生成回答      │  ← Claude API 综合所有信息回答
│       ↓              │
│  ⑤ 记忆写入         │  ← 把本次对话存入记忆库

│  用户档案（PostgreSQL）│  ← 姓名、体质、偏好

│  知识库（向量数据库）   │  ← 五行+肠脑轴+调理方案
└──────────────────────┘
```

## 五大模块详解

### 模块一：消息接入（微信怎么连上去？）

**2026 年唯一合规方案 = 企业微信 API**（个人微信机器人已被全面封杀）

```
你需要做的：
1. 注册一个企业微信（免费，不需要营业执照）
2. 创建"自建应用"
3. 配置回调 URL（系统的接收地址）
4. 把企业微信的机器人拉进你的客户群

技术实现：
- Python + FastAPI 搭建回调服务
- 接收群消息 → 解析 → 调用 AI → 回复
```

**用户体验**：用户在群里 @AI助教 + 问题 → 3-5秒内收到回复

### 模块二：用户识别 + 档案

**每个用户有一个"健康档案"**

```json
{
  "user_id": "wx_zhangsan_001",
  "name": "张三",
  "gender": "男",
  "age": 35,
  "constitution": "脾虚湿重",     // 五行体质
  "tags": ["肠胃问题", "失眠"],
  "preferences": ["不吃辣", "素食"],
  "history_summary": "3月初开始调理脾胃，目前坚持了3周",
  "alert_level": "normal",
  "created_at": "2026-03-15",
  "last_active": "2026-03-30"
}
```

**档案怎么来的？**
- 首次对话时 AI 自动引导填写基本信息
- 后续对话中 AI 自动提取关键信息更新档案
- 做完五行体质测评后自动写入

### 模块三：永久记忆（核心壁垒）

**三层记忆系统**

| 层级 | 存什么 | 用什么存 | 举例 |
|------|--------|---------|------|
| **L1 短期** | 当前对话上下文 | Redis（内存） | "他刚问了失眠的问题" |
| **L2 语义** | 历史对话片段 | 向量数据库（Qdrant） | "3天前他说最近胃胀" |
| **L3 结构** | 用户画像/事实 | PostgreSQL | "他是脾虚体质，不吃辣" |

**记忆怎么工作？**

```
用户："我最近又开始失眠了"
        ↓
系统内部处理：
  1. L3 查档案 → 体质：脾虚湿重
  2. L2 搜历史 → 找到3周前的失眠对话 + 调理方案
  3. L1 当前上下文 → "又"说明之前好过，现在复发
        ↓
AI 回答："张三你好，我记得你3周前也提到失眠，
当时建议你睡前泡脚+酸枣仁茶，后来好转了。
这次复发可能跟最近饮食有关——你脾虚体质容易
因为吃生冷导致失眠。要不要把最近的饮食告诉我，
我帮你具体分析一下？"
```

> **这就是"有记忆的 AI"和"没记忆的 ChatGPT"的本质区别。**

### 模块四：知识库（RAG）

**把你的专业知识喂给 AI**

| 知识来源 | 内容 | 格式 |
|----------|------|------|
| 五行体质 | 五种体质特征+调理方案 | Markdown/PDF |
| 肠脑轴 | 研究笔记、科普内容 | NotebookLM 导出 |
| 调理方案 | 食疗/作息/心理调节 | 结构化模板 |
| 产品知识 | 益生菌/保健品 | 产品文档 |
| 常见问题 | FAQ 集合 | Q&A 对 |

**技术实现**：
```
文档 → 切片 → 向量化（Embedding）→ 存入 Qdrant
用户提问 → 向量化 → 在 Qdrant 中搜索最相关的知识片段
知识片段 + 用户档案 + 对话历史 → 一起发给 Claude → 生成回答
```

### 模块五：主动推送 + 专家转介

**不只是被动回答，还会主动关怀**

```
每天早上 8:00：
  遍历所有活跃用户 →
  根据每人体质 + 当天节气 →
  AI 生成个性化晨起建议 →
  推送到群里 @用户

例如：
  "@张三 早安！今天谷雨，湿气重，
   你脾虚体质要特别注意：
   1. 早餐加一碗薏米粥
   2. 避免冷饮
   3. 午后可按揉足三里5分钟"
```

**专家转介**：
```
如果 AI 判断问题超出能力范围（如：用户描述急性腹痛）：
  → AI 回复："这个情况我建议你咨询专业医生"
  → 同时私信管理员："⚠️ 张三描述急性腹痛，请关注"
```

## 技术栈总览

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 语言 | **Python 3.12** | 全栈统一 |
| Web框架 | **FastAPI** | 接收企微回调 |
| AI 大模型 | **Claude API** | 回答质量最高 |
| 向量数据库 | **Qdrant**（开源免费） | 存对话记忆+知识库 |
| 关系数据库 | **PostgreSQL** | 存用户档案 |
| 缓存 | **Redis** | 短期记忆+会话状态 |
| 记忆框架 | **Mem0** | 自动提取/更新用户画像 |
| 定时任务 | **APScheduler** | 每日推送 |
| 部署 | 云服务器（腾讯云） | ¥50-100/月 |
