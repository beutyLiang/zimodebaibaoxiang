## 🔄 协作流程

### 新产品从 0 到上架

```
知远调研（市场+竞品+痛点）
    ↓
明哲判断（做不做+定价+文案方向）
    ↓
Zoey 拍板
    ↓
子默执行（写内容+做PDF+上架Dodo+生成链接）
    ↓
明哲审核（英文文案最终确认）
    ↓
Zoey 发布（推文+Reddit+YouTube）
    ↓
知远追踪（数据分析+下一步建议）
```

### 日常运营

```
每天：子默提醒 Zoey → Zoey 执行发布 → 明哲审核英文内容
每周：知远出数据报告 → 明哲做战略调整 → 子默执行
每月：四人复盘（数据+方向+下月计划）
```

### 📊 决策日报（Decision Log）

> **知远提议，3/20 正式启用**
> 目的：给每个重要决策加"数据审计"，减少拍脑袋

**流程：**
```
子默/明哲 提出建议
        ↓
子默自动准备"知远审核文本"（附在建议后面）
        ↓
Zoey 复制给知远
        ↓
知远 200 字回报：
  - 数据支持度：0-100%
  - 潜在风险点
  - 判断：做 / 不做 / 再等等
        ↓
Zoey 最终拍板
```

**子默规则：** 每次提出重要建议后，必须主动附上一段话给 Zoey 复制发知远：
```
知远，子默/明哲建议如下：
[建议内容]
请用 200 字回报：数据支持度（0-100%）+ 潜在风险点 + 你的判断（做/不做/再等等）
```

**什么情况需要走决策日报：**
- 新产品方向
- 定价变动
- 新平台/新渠道
- 花钱的决定
- 改战略方向

**什么情况不需要：**
- 日常发推文/小红书
- 写文案
- 技术执行
