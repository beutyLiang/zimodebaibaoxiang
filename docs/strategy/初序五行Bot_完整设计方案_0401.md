## 三、5道题 — 对话式设计

### 设计原则
- 每道题 = 一段聊天，不是问卷
- 选项用 emoji + 短句，不用 A/B/C
- **每次用户回答后，Bot 先回应她**（"嗯，辛苦了"），再问下一个
- 语气：像朋友，不像医生

---

### 🟢 木型体质 — 「心里装了太多事的人」

```
🌿 你是「木型」体质

你是一个心里装了太多事的人。

总觉得自己还不够好，
总想再多做一点，
但身体已经在说：慢一点。

木型的人，
肝气容易不畅，
所以你容易头痛、失眠、太阳穴胀。
不是你身体差，
是你承担的太多了。

🌙 给你的 3 件小事：

1. 睡前按一按脚背的「太冲穴」（3分钟就够）
2. 每天花15分钟做让你开心但没用的事
3. 馋酸的就吃——酸入肝，是身体在帮自己调节

你已经很努力了。
允许自己休息，不是放弃，是充电 🌿
```

## 七、Coze 工作流设计

```
步骤1：开场白节点
  → 展示欢迎语 + 按钮

步骤2：问题循环节点（5次循环）
  → 展示问题 → 等待用户选择 
  → 记录答案（对应五行+1分）
  → 发送温暖回应 → 进入下一题

步骤3：计分节点（代码节点）
  → 统计 5 题的五行得分
  → 取最高分为"主体质"
  → 并列时取 Q1 为主

步骤4：生成结果节点（大模型节点）
  → 根据主体质类型
  → 输出对应的结果文案

步骤5：生成海报节点（图片生成插件）
  → 将体质名称+扎心文案+品牌信息
  → 合成一张分享海报

步骤6：导流节点
  → 展示企微二维码/链接
  → "想要完整方案，加好友领取"
```

## 八、Prompt 模块化设计（学 Claude Code 的思路）

### 模块1：角色设定

```
你是"初序"，一个温暖的体质顾问。
你不是医生，不做诊断。
你是一个能读懂身体语言的朋友。

你的语气规则：
- 像朋友聊天，不像医生问诊
- 每次用户回答后，先回应她的感受，再问下一个问题
- 用"你"不用"您"
- 用短句，不用长段落
- 多用"嗯"、"辛苦了"、"你已经很努力了"
- 绝对不说"建议你去医院"这类话
```

### 模块2：知识库（五行体质数据）

```
[独立管理的知识库文件]
包含：5种体质的完整描述、调养建议、对应食物、穴位等
```

### 模块3：输出格式

```
结果输出要求：
1. 先说「你是XX型体质」
2. 用2-3句话描述"你是什么样的人"（被理解的感觉）
3. 再用1句话说"你的身体在说什么"
4. 最后给3件具体可执行的小事
5. 结尾一句温暖的话
```

## 九、审核风险预判

| 风险点 | 应对措施 |
|--------|---------|
| 医疗合规 | 全文不出现"治疗""诊断""处方"，只说"了解身体状态" |
| 导流合规 | 不放个人微信码，只用企微码或公众号 |
| 内容审核 | 不涉及政治/暴力/色情，纯健康生活方式 |
| 隐私合规 | 不收集姓名/手机号/身份证，只做匿名测评 |
