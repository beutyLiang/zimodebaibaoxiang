# AI 团队交接协议 v1.0

> 灵感来源：Claude Code 泄露源码中的 Coordinator Mode（task-notification 协议 + 共享暂存目录）

## 使用方法

1. 任务发起方（通常是子默）创建一个交接文件，命名格式：`YYYYMMDD_任务简述_to_接收者.md`
2. Zoey 把文件内容**整块复制粘贴**给接收方 AI，只需说一句："按这个执行"
3. 接收方完成后，Zoey 把结果粘贴回发起方（或直接存入交付目录）
4. Zoey 全程**不需要翻译、解释、补充背景**

## 团队角色

| 角色 | AI | 平台 | 职责 |
|------|-----|------|------|
| COO/CTO | 子默 | Claude (Antigravity) | 策略、技术、架构、审核 |
| CMO | 明哲 | ChatGPT | 文案、内容、营销执行 |
| CRO | 知远 | Gemini | 数据调研、商业验证 |
| CSO | 知微 | DeepSeek | 交叉验证、战略补充 |

## 交接文件模板

```markdown
# 任务交接单

## 📤 来源
- 发起人：（谁写的这个交接单）
- 时间：（日期）

## 📥 交给谁
- 接收人 + 角色

## 🎯 任务背景
（1-2句话，为什么要做这个，不超过3行）

## ✅ 已完成的工作
（前置工作，附文件路径或关键结论）

## 📋 需要你做什么
（明确、具体、可执行的指令列表，不是"帮我看看"）

## 🚫 关键约束
（必须遵守的底线）

## 📦 交付物
（完成后应该输出什么，格式要求）
```
